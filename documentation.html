Documentation

Welcome to the Salesforce User Inspector documentation. This extension helps you analyze and manage Salesforce users with ease.

Features

Salesforce User Inspector provides a comprehensive set of tools for analyzing user permissions and access:

  • Inspect Permission Set and Permission Set Group assignments
  • View and manage Public Group memberships
  • Manage Queue assignments and access
  • Analyze object and field-level permissions along with their source

Getting Started

Installation

Install the extension from the Chrome Web Store. Once installed, click the extension icon in your browser toolbar to open the inspector.

Using the Extension

To use this extension, you must create a Connected App in Salesforce. Please follow the steps below:

  • Create a Connected App with any name and contact email of your choice.
  • Enable OAuth Settings.
  • Set the Callback URL in the format: 'https://-extension_ID-.chromiumapp.org/auth/callback'.
  • Select the OAuth scope: “Manage user data via APIs (api)”.
  • Under security settings, disable “Require Proof Key for Code Exchange (PKCE) extension for Supported Authorization Flows.”
  • Save the Connected App configuration.
  • Navigate to the Consumer Key and Secret section. Complete email verification if prompted, and securely store the generated Consumer Key.

After installing the extension:

  • Open the extension and click “+ Add/Remove Configuration.”
  • Provide a configuration name of your choice.
  • Enter the Login URL (preferably your organization-specific Salesforce domain instead of "login.salesforce.com").
  • Add the Consumer Key generated from your Connected App.
  • Click Save to store the configuration.
  • From the dropdown menu, select the desired organization configuration from the saved entries and click “Login with Salesforce.”
  • To delete an existing configuration, select the relevant organization from the dropdown, click “+ Add/Remove Configuration,” and then choose “Delete” to remove it.
